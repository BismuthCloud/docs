## `cli`

The CLI for Bismuth Cloud

**Usage:** `cli [OPTIONS] <COMMAND>`

###### **Subcommands:**

- `login` — Login to Bismuth Cloud
- `version` — Show the CLI version
- `project` — Manage projects
- `feature` — Manage features
- `kv` — Interact with key-value storage
- `blob` — Interact with blob (file) storage
- `sql` — Run SQL queries against a feature's database
- `billing` —
- `import` — Create a new Bismuth project, and import an existing Git repository into it. Alias of `project import`
- `deploy` — Deploy a feature to the cloud. Alias of `feature deploy`
- `deploy-status` — Get the status of a deployment. Alias of `feature deploy-status`
- `teardown` — Teardown a feature. Alias of `feature teardown`
- `get-url` — Get the URL for a deployed feature. Alias of `feature get-url`
- `logs` — Get logs from a deployment. Alias of `feature logs`
- `chat` — Interact with the Bismuth AI

###### **Options:**

- `--api-url <API_URL>`

  Default value: `https://api.bismuth.cloud`

- `--config-file <CONFIG_FILE>`

  Default value: `/Users/crow/Library/Application Support/bismuth.json`

- `-v`, `--verbose` — Increase logging verbosity
- `-q`, `--quiet` — Decrease logging verbosity
